# UI v1 (Spec D) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans.

**Goal:** A single-page operator UI that exercises the v2 backend surface, bundled into the `prioritarr-app` container.

**Spec:** `docs/specs/2026-04-18-prioritarr-ui-design.md`

**Stack:** React 19 + TS 5 + Vite 5 + TanStack Query/Router/Table + Tailwind v4 + Radix UI primitives + `@microsoft/fetch-event-source` for SSE.

---

## Critical pre-flight

1. Spec C has merged to main; `ghcr.io/cquemin/prioritarr-app:latest` is built from the new Ktor backend with `/api/v2/*`.
2. This plan lives on `feat/spec-d-ui` (branched from main).
3. The UI is write-last — no task here touches the backend. If a task ever thinks it needs a backend change, stop and open a separate Spec C.1.

## Task 2: API client via openapi-typescript

**Files:**
- Create: `src/api/generated.ts` (codegen output)
- Create: `src/api/client.ts`
- Modify: `package.json` (add `gen:api` script)

**Steps:**
- [ ] `pnpm add -D openapi-typescript openapi-fetch`.
- [ ] `pnpm gen:api` runs `openapi-typescript ../../openapi.json -o src/api/generated.ts` (paths relative to `frontend/`).
- [ ] `client.ts` wraps `openapi-fetch` with:
  - `baseURL: import.meta.env.VITE_BACKEND_URL ?? ''`
  - auth-injecting middleware that reads `localStorage.prioritarr_api_key`, sets `X-Api-Key` on every request, and throws `Unauthorized` on 401.
- [ ] Smoke-test at runtime: `const { data } = await client.GET('/api/v2/settings')` in the console, confirms 401 without a key, 200 with one.
- [ ] Commit: `feat(ui/api): openapi-typescript client + auth middleware`

## Task 3: Auth flow + login route

**Files:**
- Create: `src/hooks/useAuth.ts`, `src/routes/login.tsx`

**Steps:**
- [ ] `useAuth()` hook: exposes `apiKey`, `setApiKey(k)`, `logout()`. Reads/writes `localStorage`. Subscribes to storage events so another tab's logout propagates.
- [ ] On mount, App checks `useAuth().apiKey`; if null, redirect to `/login`.
- [ ] Login page: single text input + Connect button. On submit, attempts `GET /api/v2/settings` with the entered key; 200 → persist + navigate to `/series`; 401 → show error toast.
- [ ] Add a `LogoutButton` component, used later in `/settings`.
- [ ] Unit tests: mock the client, assert login flow + the 401 case.
- [ ] Commit: `feat(ui/auth): API key login + localStorage persistence`

## Task 5: SSE hook + cache invalidation plumbing

**Files:**
- Create: `src/hooks/useEventStream.ts`

**Steps:**
- [ ] Use `@microsoft/fetch-event-source`:
  ```ts
  fetchEventSource('/api/v2/events', {
    headers: { 'X-Api-Key': apiKey, 'Last-Event-ID': lastId },
    onmessage: handleEvent,
    onclose: () => scheduleReconnect(),
    openWhenHidden: true,
  })
  ```
- [ ] Map each event type to TanStack Query invalidations per Spec §6.3.
- [ ] Persist `lastEventId` to `localStorage` on each event, replay on reconnect.
- [ ] Expose `useEventStream().status` for the live/disconnected dot in the shell.
- [ ] Unit test: pipe a scripted stream through the hook, assert invalidation keys are queued.
- [ ] Commit: `feat(ui/sse): event stream with reconnect + cache invalidation`

## Task 14: Dockerfile multi-stage: node build → kotlin jar → runtime

**Files:**
- Modify: `prioritarr-app/Dockerfile`
- Modify: `backend/src/main/kotlin/.../app/Module.kt` — add `staticResources("/", "static")` to serve the bundled UI.

**Steps:**
- [ ] Add a `FROM node:22-alpine AS ui-build` stage before the gradle stage. Runs `pnpm install --frozen-lockfile && pnpm build`. Output: `/ui/dist/`.
- [ ] Gradle build stage copies `/ui/dist/` into `backend/src/main/resources/static/` before `shadowJar`.
- [ ] Kotlin route: `route("/") { staticResources("/", "static") }` registered AFTER all `/api/*` routes so the SPA index.html is the fall-through.
- [ ] Client-side routing fallback: any non-api path that 404s in static should return `index.html` (for deep-link refreshes). Add a Ktor route-catcher.
- [ ] Build the image locally, run, confirm the UI loads at `http://localhost:8000/`.
- [ ] Commit: `feat(ui/docker): multi-stage build — node → gradle → jre runtime`

## Plan self-review notes

1. **Spec coverage:** §4 stack → Task 1; §5 layout → Task 4; §6 data flow → Tasks 2+5+6+7+8+9; §7 components → Tasks 6-10; §8 serving → Task 14; §9 openapi → Task 2 + CI drift in 15; §10 testing → Tasks 12+13; §12 acceptance → Task 17.

2. **Commit cadence:** ~17 commits, each independently pnpm-build-able. One per page keeps diff review tractable.

3. **Biggest risk:** Task 5 (SSE reconnect semantics) + Task 14 (Kotlin + static resources + SPA fallback route ordering). Both have a fast local feedback loop; budget an extra half-day if Task 14's route ordering trips over Ktor's matching rules.

4. **The build-time node dep:** adds ~350MB to the build image layer. Runtime stays eclipse-temurin JRE (no node). Acceptable.
